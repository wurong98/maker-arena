# 前端界面设计文档

## 1. 页面布局

```
┌─────────────────────────────────────────────────────────────────────────┐
│  Header: Logo + 系统统计 (策略数/成交数/挂单数)                         │
├──────────────┬────────────────────────────────────────────────────────┤
│              │  Content Header: 策略名称 + 余额 + 收益率                │
│   左侧        │────────────────────────────────────────────────────────│
│  策略排行榜   │  Tab: [持仓] [成交记录] [订单] [快照] [强平记录]        │
│              │────────────────────────────────────────────────────────│
│  (可滚动)    │                                                         │
│              │                    内容区域                              │
│              │                 (表格/图表)                              │
│              │                                                         │
├──────────────┴────────────────────────────────────────────────────────┤
│  Footer: BTCUSDC $67,234.50 +1.87% | ETHUSDC $3,456.78 -1.29% ...    │
└────────────────────────────────────────────────────────────────────────┘
```

## 2. 技术方案

| 项 | 方案 |
|---|---|
| 技术栈 | HTML + CSS + JavaScript (原生) |
| 图表库 | Chart.js (CDN) |
| 数据请求 | fetch API |
| 更新频率 | 行情 5 秒轮询，其他数据点击时加载 |

## 3. API 调用设计

| 前端需求 | API 端点 | 参数 |
|---------|---------|------|
| 获取策略列表 | GET /api/v1/strategies | page, limit |
| 获取系统统计 | GET /api/v1/statistics | - |
| 获取持仓 | GET /api/v1/exchange/getPosition | strategyId |
| 获取成交记录 | GET /api/v1/fills | strategyId, page, limit |
| 获取订单 | GET /api/v1/exchange/getOrders | strategyId, page, limit |
| 获取账户快照(余额曲线) | GET /api/v1/snapshots/account | strategyId |
| 获取持仓快照(持仓价值曲线) | GET /api/v1/snapshots/position | strategyId |
| 获取强平记录 | GET /api/v1/liquidations | strategyId, page, limit |
| 获取行情 | GET /api/v1/market/ticker | - |

## 4. 组件设计

### 4.1 策略排行榜 (左侧)

| 列 | 说明 |
|---|---|
| 排名 | #1, #2, #3... |
| 策略名 | 策略名称 |
| 余额 | 当前余额 |
| 收益率 | 百分比，绿色/红色区分涨跌 |

交互：点击行选中策略，高亮显示

### 4.2 持仓 Tab

| 列 | 说明 |
|---|---|
| 交易对 | btcusdc |
| 方向 | 多(绿色) / 空(红色) |
| 数量 | 持仓数量 |
| 开仓价 | 入场价格 |
| 当前价 | 当前市场价格（从 ticker 接口获取） |
| 强平价 | 强平触发价格 |
| 未实现盈亏 | 绿色/红色 |

### 4.3 成交记录 Tab

| 列 | 说明 |
|---|---|
| 时间 | 成交时间 |
| 交易对 | 交易对 |
| 方向 | 买入(绿色) / 卖出(红色) |
| 价格 | 成交价格 |
| 数量 | 成交数量 |
| 手续费 | 手续费金额 |

分页：每页 20 条

### 4.4 订单 Tab

| 列 | 说明 |
|---|---|
| 时间 | 下单时间 |
| 交易对 | 交易对 |
| 类型 | 限价/市价 |
| 方向 | 买入/卖出 |
| 价格 | 挂单价格 |
| 数量 | 挂单数量 |
| 已成交 | 已成交数量 |
| 状态 | 挂单中/已成交/已取消/已强平 |

### 4.5 快照 Tab

- **收益曲线图**: 使用 `/api/v1/snapshots/account` 的 `totalEquity` 字段计算收益
- **余额曲线图**: 使用 `/api/v1/snapshots/account` 的 `balance` 字段
- **持仓价值曲线**: 使用 `/api/v1/snapshots/position` 的 `positionValue` 字段
- 时间范围：默认显示最近 24 小时，无时间范围选择器

### 4.6 强平记录 Tab

| 列 | 说明 |
|---|---|
| 时间 | 强平时间 |
| 交易对 | 交易对 |
| 方向 | 多/空 |
| 强平价格 | 触发价格 |
| 数量 | 强平数量 |

### 4.7 底部行情栏

| 列 | 说明 |
|---|---|
| 交易对 | BTCUSDC |
| 价格 | 最新价格 |
| 24h 涨跌 | 百分比，绿色/红色 |
| 更新时间 | HH:mm:ss |

## 6. 响应式设计

- 最小宽度支持 1024px
- 左侧边栏固定宽度 320px
- 内容区域自适应
- 表格横向滚动

## 7. 交互逻辑

1. 页面加载 → 获取策略列表 + 获取行情
2. 点击左侧策略 → 加载该策略的持仓/成交/订单/快照/强平记录
3. 切换 Tab → 切换显示内容
4. 底部行情每 5 秒轮询更新
